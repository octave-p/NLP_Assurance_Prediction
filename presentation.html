Supervised ML — 3 Architecture Comparison

Sentiment Analysis

3-class classification: Négatif (≤2★) · Neutre (3★) · Positif (≥4★)

Architecture Accuracy Macro F1 F1 Négatif F1 Neutre F1 Positif
TF-IDF + Logistic Regression PROD 0.754 0.666 0.83 0.30 0.80
CamemBERT (sentence-transformers) 0.725 0.643 0.83 0.30 0.80
Keras (custom Embedding layer) 0.662 — — — —

TF-IDF + LR

max_features=10K, ngram (1,2), sublinear_tf. Captures polarized vocabulary extremely well. Class weights balanced.

→ Selected for production

CamemBERT

dangvantuan/sentence-camembert-base — 384-dim embeddings via mean pooling. No fine-tuning on domain data.

→ Loses contextual edge without FT

Keras Deep Learning

50-dim Embedding → GlobalAvgPool1D → Dense(32) → Dropout(0.5) → Dense(3, softmax). 10 epochs, batch 64.

→ Insufficient training volume
Why TF-IDF wins: Insurance reviews are highly polarized with explicit signal words ("arnaque", "parfait", "nul"). TF-IDF captures this vocabulary directly. A Transformer without fine-tuning cannot leverage its contextual advantage on this domain.

Conclusion & Key Takeaways

Summary

Data Engineering

  • 34,435 French insurance reviews cleaned end-to-end
  • Domain-aware stopwords preserved semantic integrity
  • Spell-checker consciously excluded to protect proper nouns
  • English translation export for future model compatibility

Unsupervised Learning

  • Word2Vec captured business semantics: prix↔tarif (0.078)
  • Cosine distance validated insurance domain vocabulary clusters
  • NMF topic modeling structured review themes automatically

Model Comparison — Key Insight

  • TF-IDF + LR: 75.4% accuracy, F1 0.667 → PRODUCTION
  • CamemBERT (no FT): 72.5% — contextual power lost without fine-tuning
  • Keras custom embeddings: 66.2% — insufficient data volume
  • Polarized vocabulary rewards statistical frequency methods

Production RAG System

  • 4-page Streamlit app — fully local, zero external dependencies
  • Extractive RAG: entity + intent detection → database retrieval
  • 100% factual — answers grounded in real customer verbatims
  • Near-zero inference latency via cached models & TF-IDF pipeline
Classical ML still competitive — domain vocabulary matters more than model complexity
